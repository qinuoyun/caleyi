## 二、模型层规范 (modules/)

### 2.1 基础模型定义

**必须** 使用 `ci.Model` 作为基础模型，框架自动管理以下字段：
- `ID` - 主键
- `TenantID` - 租户ID
- `CreatedAt` - 创建时间
- `UpdatedAt` - 更新时间
- `DeletedAt` - 删除时间（软删除）

```go
// ✅ 正确写法
type Expert struct {
    ci.Model                                                    // 必须嵌入
    AccountID int64  `gorm:"column:account_id;index" json:"account_id"`
    Name      string `gorm:"column:name;type:varchar(100)" json:"name"`
    Status    int    `gorm:"column:status;default:1" json:"status"`
}

// ❌ 错误写法 - 不要自己定义这些字段
type Expert struct {
    ID        uint   `gorm:"primaryKey"`
    TenantID  string `gorm:"index"`
    CreatedAt time.Time
    // ...
}
```

### 2.2 表名定义

```go
func (Expert) TableName() string {
    return utils.GetTableName(Expert{})
}
```

### 2.3 模块初始化

```go
func init() {
    path := Expert{}
    ci.BinModule(&path)
}
```

### 2.4 数据隔离

所有与Account账号有关的业务数据模型 **必须** 包含 `AccountID` 字段实现账号级隔离：

```go
type Expert struct {
    ci.Model
    AccountID int64 `gorm:"column:account_id;index" json:"account_id"` // 必须
    // ...
}

type Tag struct {
    ci.Model
    AccountID int64 `gorm:"column:account_id;index" json:"account_id"` // 必须
    // ...
}

type Knowledge struct {
    ci.Model
    AccountID int64 `gorm:"column:account_id;index" json:"account_id"` // 必须
    // ...
}
```

## 三、数据库操作规范

### 3.1 同步操作（控制器中）

在控制器中使用 `ci.M("模型名")` 进行数据库操作：

```go
func (con ExpertController) Index(c *gin.Context) {
    // ci.M() 参数为模型名称（小写）
    ci.M("expert").Where("account_id = ?", accountID).Find(&list)
    ci.M("account").Where("id = ?", id).First(&account)
    ci.M("tag").Create(&tag)
}
```

### 3.2 模型方法（两种写法）

**方式一：简单写法（适用于同步场景）**

```go
// 模型不需要定义 Create/Update 方法，直接在控制器中调用
ci.M("expert").Create(&expert)
ci.M("expert").Where("id = ?", id).Updates(&expert)
ci.M("expert").Delete(&expert)
```

**方式二：带 tenantID 参数（适用于异步场景）**

```go
// 模型定义带 tenantID 的方法
func (m *Expert) Create(tenantID string) error {
    return ci.MT(tenantID, &Expert{}).Create(m).Error
}

func (m *Expert) Update(tenantID string) error {
    return ci.MT(tenantID, &Expert{}).Where("id = ?", m.ID).Updates(m).Error
}

// 在异步任务中调用
go func(tenantID string) {
    expert.Create(tenantID)
}(tenantID)
```

**注意**：不要手动设置 `m.TenantID = xxx`，框架会自动处理

### 3.3 异步操作（goroutine 中）

在异步任务中 **必须** 使用 `ci.DBWithTenant(tenantID)`：

```go
// ✅ 正确写法
func processAsync(tenantID string, data SomeData) {
    go func() {
        db := ci.DBWithTenant(tenantID)
        db.Create(&record)
    }()
}

// ❌ 错误写法 - gin.Context 在异步中不可用
func processAsync(c *gin.Context, data SomeData) {
    go func() {
        db := ci.GetDB(c)  // 会报错：tenant ID not found in context
        db.Create(&record)
    }()
}
```

### 3.4 获取 TenantID

```go
// 方式1：从 gin.Context 获取
tenantID := ci.GetTenantID(c)

// 方式2：从 Query 参数获取（备用）
if tenantID == "" {
    tenantID = c.Query("tenant_id")
}
```

## 四、控制器规范 (controllers/)

### 4.1 控制器命名

**控制器必须使用 `名称 + Controller` 后缀命名**

```go
// ✅ 正确写法
type AccountController struct{}
type ExpertController struct{}
type TagController struct{}
type KnowledgeController struct{}

// ❌ 错误写法
type Expert struct{}      // 缺少 Controller 后缀
type ExpertCtrl struct{}  // 后缀不规范
type Experts struct{}     // 缺少 Controller 后缀
```

### 4.2 控制器定义

```go
package expert

import (
    "reflect"
    "github.com/gin-gonic/gin"
    "github.com/qinuoyun/caleyi/utils/ci"
)

type ExpertController struct{}

func init() {
    path := ExpertController{}
    ci.BinController(&path, reflect.TypeOf(path).PkgPath())
}
```

### 4.3 获取当前登录账号ID

使用框架内置方法：

```go
accountID := ci.GetAccountID(c)
```

### 4.4 账号数据隔离（仅限账号相关业务）

**仅与 Account 账号有关的业务** 需要做账号隔离和数据过滤，无关业务不需要。

**需要账号隔离的模型**：
- `Expert` - AI专家（属于某个账号）
- `Tag` - 标签（属于某个账号）
- `Knowledge` - 知识库（属于某个账号）
- `Contact` - 联系人（属于某个账号）
- `Conversation` - 会话（属于某个账号）

**不需要账号隔离的模型**：
- `Dict` - 字典（全局共享）
- `Role` - 角色（全局共享）
- `Question` - 题目（全局共享）
- 其他全局配置类模型

**列表查询过滤账号**：

```go
// ✅ 需要账号隔离的业务
func (con ExpertController) Index(c *gin.Context) {
    accountID := ci.GetAccountID(c)
    
    search := make(map[string]interface{})
    search["account_id"] = accountID  // 必须过滤
    
    list, total, err := Expert.GetExpertList(c, page, pageSize, search)
}

// ✅ 不需要账号隔离的业务
func (con DictController) Index(c *gin.Context) {
    search := make(map[string]interface{})
    // 无需 account_id 过滤
    
    list, total, err := Dict.GetDictList(c, page, pageSize, search)
}
```

### 4.5 创建数据设置账号（仅限账号相关业务）

**同步操作**：

```go
// ✅ 需要账号隔离的业务 - 必须设置 AccountID
func (con ExpertController) Create(c *gin.Context) {
    accountID := ci.GetAccountID(c)
    
    expert := Expert.Expert{
        AccountID: accountID,  // 必须设置
        Name:      req.Name,
    }
    
    if err := ci.M("expert").Create(&expert).Error; err != nil {
        ci.Error(c, 50001, "创建失败")
        return
    }
}

// ✅ 不需要账号隔离的业务 - 无需设置 AccountID
func (con DictController) Create(c *gin.Context) {
    dict := Dict.Dict{
        Name: req.Name,
        // 无需 AccountID
    }
    
    if err := ci.M("dict").Create(&dict).Error; err != nil {
        ci.Error(c, 50001, "创建失败")
        return
    }
}
```

**异步操作（goroutine 中）**：需要先获取 `tenantID`

```go
func (con ExpertController) CreateAsync(c *gin.Context) {
    accountID := ci.GetAccountID(c)
    tenantID := ci.GetTenantID(c)  // 异步前获取
    
    go func(tenantID string, accountID int64) {
        expert := Expert.Expert{
            AccountID: accountID,
            Name:      "xxx",
        }
        ci.MT(tenantID, &Expert.Expert{}).Create(&expert)
        // 或
        ci.DBWithTenant(tenantID).Create(&expert)
    }(tenantID, accountID)
}
```

## 九、代码风格

### 9.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | 小写单词 | `expert`, `knowledge` |
| 结构体 | 大驼峰 | `ExpertController`, `ChatService` |
| 变量 | 小驼峰 | `accountID`, `tenantID` |
| 常量 | 大驼峰或全大写 | `FixedModule`, `MAX_SIZE` |
| 数据库字段 | 下划线 | `account_id`, `created_at` |
| JSON字段 | 下划线 | `"account_id"`, `"tenant_id"` |

### 9.2 方法命名规范（重要）

**控制器、模型、服务的方法名必须是单个单词，禁止多单词组合**

```go
// ✅ 正确写法 - 单个单词
func (con ExpertController) Index(c *gin.Context)   // 列表
func (con ExpertController) Detail(c *gin.Context)  // 详情
func (con ExpertController) Create(c *gin.Context)  // 创建
func (con ExpertController) Update(c *gin.Context)  // 更新
func (con ExpertController) Delete(c *gin.Context)  // 删除
func (con ExpertController) Options(c *gin.Context) // 选项
func (con ExpertController) All(c *gin.Context)     // 全部

func (m *Expert) GetByID(id int64) error  // 模型方法例外，可用于内部
func (s *ChatService) Send(msg string)    // 服务方法

// ❌ 错误写法 - 多单词组合
func (con ExpertController) GetList(c *gin.Context)      // 错误
func (con ExpertController) CreateExpert(c *gin.Context) // 错误
func (con ExpertController) GetDetail(c *gin.Context)    // 错误
func (con ExpertController) UpdateStatus(c *gin.Context) // 错误
```

**常用控制器方法名**：
- `Index` - 列表
- `Detail` - 详情
- `Create` - 创建
- `Update` - 更新
- `Delete` - 删除
- `All` - 全部（不分页）
- `Options` - 选项
- `Export` - 导出
- `Import` - 导入
- `Search` - 搜索
- `Status` - 状态
- `Sync` - 同步

### 9.3 GORM Tag 规范

```go
type Expert struct {
    ci.Model
    AccountID   int64  `gorm:"column:account_id;index" json:"account_id"`
    Name        string `gorm:"column:name;type:varchar(100);not null" json:"name"`
    Description string `gorm:"column:description;type:text" json:"description"`
    Status      int    `gorm:"column:status;default:1" json:"status"`
}
```

### 9.4 错误处理

```go
// 返回错误信息给调用方
if err != nil {
    return fmt.Errorf("操作失败: %v", err)
}

// 控制器中返回错误响应
if err != nil {
    ci.Error(c, 50001, "操作失败: "+err.Error())
    return
}
```

## 十一、常见问题

### Q1: tenant ID not found in context

**原因**：在异步任务中使用了 `ci.GetDB(c)`

**解决**：
```go
// 在异步任务外获取 tenantID
tenantID := ci.GetTenantID(c)

go func() {
    // 在异步任务内使用 ci.DBWithTenant
    db := ci.DBWithTenant(tenantID)
    db.Create(&record)
}()
```

### Q2: 数据没有按账号隔离

**原因**：查询时没有过滤 `account_id`

**解决**：
```go
search["account_id"] = ci.GetAccountID(c)
list, _, _ := Expert.GetExpertList(c, page, pageSize, search)
```

### Q3: 交叉编译失败 (silk 库)

**原因**：silk 库依赖 CGO，无法交叉编译

**解决**：在目标服务器上执行编译脚本
